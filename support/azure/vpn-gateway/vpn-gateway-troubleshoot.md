# Troubleshoot VPN Gateway

## Summary

VPN Gateway connections can fail for a variety of reasons. This article contains links to get you started with troubleshooting. For a full list, see the articles contained in the table of contents under **Troubleshoot**, to the left of this page.

## Troubleshooting scenarios and solutions

* [Validate VPN throughput to a VNet](/azure/vpn-gateway/vpn-gateway-validate-throughput-to-vnet)<br>A VPN gateway connection enables you to establish secure, cross-premises connectivity between your Virtual Network within Azure and your on-premises IT infrastructure. This article shows how to validate network throughput from the on-premises resources to an Azure virtual machine (VM). It also provides troubleshooting guidance.

* [VPN and Firewall device settings](/azure/vpn-gateway/vpn-gateway-about-vpn-devices)<br>This article provides several suggested solutions for third-party VPN or firewall devices that are used with VPN Gateway. Technical support for third-party VPN or firewall devices is provided by the device vendor.

* [Point-to-Site connections](/azure/vpn-gateway/vpn-gateway-troubleshoot-vpn-point-to-site-connection-problems)<br>This article lists common point-to-site connection problems that you might experience. It also discusses possible causes and solutions for these problems.

* [Site-to-Site connections](/azure/vpn-gateway/vpn-gateway-troubleshoot-site-to-site-cannot-connect)<br>After you configure a site-to-site VPN connection between an on-premises network and an Azure virtual network, the VPN connection suddenly stops working and cannot be reconnected. This article provides troubleshooting steps to help you resolve this problem.

* [Troubleshoot Azure VPN Gateway using diagnostic logs](/azure/vpn-gateway/troubleshoot-vpn-with-azure-diagnostics)<br>Using diagnostic logs, you can troubleshoot multiple VPN gateway related events including configuration activity, VPN Tunnel connectivity, IPsec logging, BGP route exchanges, Point to Site advanced logging.
